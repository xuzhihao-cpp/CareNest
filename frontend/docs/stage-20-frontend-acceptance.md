## 范围

本记录只覆盖 `frontend` 中的阶段 20 病历资料页面、上传流程和前端校验。对象存储、数据库写入、孤立文件清理及真实 `403/422/500` 响应需在阶段 20 后端接口部署后进行联调验收。

## 契约约定

- 上传文件：`POST /api/v1/files`，成功响应必须包含非空 `fileId`。
- 登记资料：`POST /api/v1/elders/{elderId}/medical-files`。
- 刷新列表：`GET /api/v1/elders/{elderId}/medical-files`。
- 阶段文档的待审核状态为 `PENDING_REVIEW`；为兼容仓库全局字典，读取时也接受 `PENDING`，并统一显示为“待审核”。
- 阶段文档的需补充状态为 `NEED_MORE`；读取时兼容 `NEEDS_SUPPLEMENT`，并统一显示为“需补充”。
- 前端不持久化 `fileId`。文件已上传但登记失败时，当前页面只能重新登记，不能更换、移除文件或切换长辈。
- 同源受保护文件通过 Bearer Token 获取后预览；跨域或带签名参数的临时授权 URL 直接打开。

## 页面检查

- 家属端可选择长辈、资料类别、文件、标题和发生日期。
- 家属端显示上传进度、登记失败重试、审核状态筛选和处理意见。
- 长辈端只显示本人病历资料，不显示上传入口。
- 390px 宽度下页面无水平溢出，长标题和四类审核状态可正常换行显示。
- 页面不显示 `fileId`、对象存储 key、接口路径、DTO 或 traceId。
- 没有删除、替换、分享等未冻结接口的按钮。
- 阶段 20 新增代码中没有 mock、Base64 文件持久化、本地假记录或假成功回退。

## 后端联调门禁

以下项目在真实后端可用后逐项执行并补充响应与数据库证据：

1. 上传成功后对象存储存在文件，`file_asset` 与 `medical_file` 关联正确，初始状态为待审核。
2. 上传超限、错误类型、上传中断、登记失败返回冻结的业务状态码，页面提示与重试路径正确。
3. 非授权家属、跨长辈访问和护理人员访问未批准资料返回 `403`。
4. 登记失败产生的孤立文件由后端删除、标记或进入清理任务。
5. 管理端审核后，家属端和长辈端刷新能读取数据库中的最新状态与处理意见。
6. 真实预览和下载地址按授权策略可访问，过期后页面显示重新刷新提示。
