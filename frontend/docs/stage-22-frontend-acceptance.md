# 阶段 22 前端验收说明

## 实现范围

- 长辈端通过大号按钮记录疼痛、头晕、睡眠、饮食和精神状态，并选择轻微、明显或严重。
- 支持快捷、文字、录音和已有语音文件四种交互，提交契约统一为 `BUTTON`、`TEXT` 或 `VOICE`。
- 家属端只对已生效且包含健康查看权限的绑定读取时间线，支持类型、程度、日期和分页筛选。
- 严重反馈仅显示联系家属和寻求线下医疗帮助的风险提示，不输出诊断、用药建议或虚假通知结果。
- 家属切换长辈时立即清空旧时间线，并以请求序号和当前长辈资源双重校验响应，旧请求不得覆盖新页面。
- 同源受保护语音由前端携带登录令牌读取为 Blob；跨域签名语音只允许来自 `VITE_MEDIA_TRUSTED_ORIGINS` 中配置的 HTTPS origin，并且绝不附带登录令牌。
- `/pages/elder/index?view=emergency` 已对应真实协助视图，提供家属联系方式、部署环境配置的平台热线和 120 急救电话操作，不自行拼接未冻结的协助提交接口。

## 真实接口门禁

1. 长辈提交仅调用 `POST /api/v1/elder/health-feedback`，请求体不得包含 `elderId`。
2. 家属读取仅调用 `GET /api/v1/family/elders/{elderId}/health-feedback`。
3. 本地开发代理将 `/api/v1/elder` 和 `/api/v1/family/elders` 转发到用户服务，不得误发到护理服务。
4. 语音先通过 `POST /api/v1/files` 上传，再将真实 `fileId` 随反馈提交。
5. 语音上传成功、反馈登记失败时保留当前会话内的 `fileId`，重新提交不得重复上传。
6. 任一接口返回 `401`、`403`、`422`、`500` 或异常结构时显示真实失败，不生成本地记录。
7. 家属语音源地址不得直接赋给播放器；鉴权读取失败时只显示不可播放状态。
8. 快速切换两位长辈并人为延迟前一个请求，页面必须始终只显示当前长辈的记录。
9. 未列入可信 origin 的跨域语音必须被拒绝；可信签名地址直接交给播放器，不得进入携带 Token 的请求流程。
10. 使用 `view=emergency` 打开长辈端时必须展示协助页面，不能回到默认档案。

## 联调检查

- 分别使用长辈账号提交五种反馈、三级程度和三种输入类型。
- 使用具有 `ACTIVE + HEALTH_VIEW` 的家属绑定确认记录可见；待确认、已撤销或缺少权限的绑定不得请求列表。
- 验证家属只能读取路径中的目标长辈资源，越权访问由后端返回 `403`。
- 验证语音文件扩展名与 MIME 一致、大小不超过 12 MB、录音不超过 60 秒；后端继续校验文件归属。
- 验证严重记录使用醒目但克制的样式，并且页面不存在诊断、调整用药或“已通知”类承诺。
