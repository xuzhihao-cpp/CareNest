# 阶段31 服务前注意事项与确认前端验收记录

## 验收依据

- 项目工作目录：`smart-nursing-platform/`
- 阶段任务卡：`docs/team/phase-26-31-optimized-ai-task-cards.md`

- 边界：前端确认仅用于服务前核对，护理任务状态仍由后端接口和数据库状态机最终裁决。

## 前端实现

### 护理端

- 阶段31嵌入阶段25服务前健康摘要，不增加独立测试页面。
- 读取当前订单的真实注意事项，并按“高风险、重点、提示”分组展示。
- 来源统一显示为“健康档案、审核通过的病历资料、本次服务要求、本次服务安排”，不展示英文枚举、内部编号或接口字段名。
- `requiredAck=true` 且尚未确认的事项提供复选框；已确认事项只读展示确认状态和时间。
- 只有具备 `NURSE_ATTENTION_ACK` 权限时才开放确认操作。
- 确认成功后重新读取注意事项，不用提交响应直接冒充最终状态。
- 任务状态为“正在前往”、注意事项读取成功且全部必确认事项均已确认后，才开放“开始服务”。
- 开始服务仍调用真实任务状态接口；后端返回 `409` 或 `422` 时重新读取任务和注意事项，不在前端强行进入服务中。
- 切换任务、刷新任务或组件卸载时调用 `AbortController.abort()` 中止旧读取与确认请求，同时使旧请求序号失效；已完成但迟到的响应也不能覆盖当前任务。
- 确认时间统一转换为 `Asia/Shanghai` 后显示；带 `Z`、时区偏移或无偏移的冻结格式均不会直接截断误读。

### 管理端与客服端

- 管理员和客服均必须具备 `CARE_ATTENTION_REVIEW` 权限才挂载审阅区域。
- 具备权限的客服可从“服务前审阅”进入订单列表，默认显示全部可审阅状态并排除待派单、已取消订单，不会获得派单、取消或改期能力。
- 审阅区域只读展示注意事项、来源、风险级别和确认状态，不提供确认复选框或状态写入按钮。
- 缺少权限或权限读取失败时关闭入口，并显示业务化提示，不发起越权读取。

## 接口与契约

- `GET /api/v1/nurse/orders/{orderId}/attention-notices`
  - 响应：`{items:[{noticeId,level,content,source,requiredAck,acknowledged,acknowledgedAt}]}`
- `POST /api/v1/nurse/orders/{orderId}/attention-notices/ack`
  - 请求：`{noticeIds}`
  - 响应：最新注意事项列表。
- `GET /api/v1/auth/permissions`
  - 护理端确认权限：`NURSE_ATTENTION_ACK`
  - 管理端与客服端审阅权限：`CARE_ATTENTION_REVIEW`

接口响应会校验事项编号、正文、风险级别、来源、确认字段和重复编号。响应不完整时显示维护提示，不生成本地补偿数据。

## 无 Mock 约束

- 阶段31 API、组件和测试均没有运行时 mock、fixture 回退或本地假成功。
- 阶段31承载的阶段13任务 API 已移除 `isMockEnabled()`、本地任务合成和 mock 参数，只读取真实后端任务。
- 自动化测试中的拦截响应仅用于测试适配器和页面行为，不进入生产构建。
