# Phase 01 验收记录：项目目录与协作规范

## 阶段目标

建立仓库目录、分支、提交信息和资料目录规范，提供后续所有阶段的协作入口。

## 验收清单

- [ ] `README.md` 可见项目定位、目录说明、分支规则和验收规则。
- [ ] `CONTRIBUTING.md` 可见多人协作规则。
- [ ] `.github/pull_request_template.md` 可见 PR 检查项。
- [ ] `.github/ISSUE_TEMPLATE/stage-task.md` 可见阶段 Issue 模板。
- [ ] `.github/CODEOWNERS` 可见默认责任路径。
- [ ] 固定目录均已创建并有说明文件。

## 证据记录

| 证据类型 | 路径或说明 | 验收人 | 结果 |
| --- | --- | --- | --- |
| 文件树 | 运行 `git status --short` 和目录检查 |  |  |
| 协作规则 | `README.md`, `CONTRIBUTING.md` |  |  |
| GitHub 模板 | `.github/` |  |  |

## 备注

阶段 1 不实现业务代码，不启动后端或前端工程。
