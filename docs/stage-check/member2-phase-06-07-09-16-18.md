## 阶段 16 状态

成员 1 已在 `db/schema/phase-16-report-ack-schema.sql` 提供 `care_report_ack`、`health_info_review_task`，阶段 16 已按数据库字段实现：

| Method | Path |
| --- | --- |
| POST | `/api/v1/elder/reports/{reportId}/ack` |
| POST | `/api/v1/family/reports/{reportId}/ack` |
| POST | `/api/v1/family/reports/{reportId}/archive-suggestions/decision` |

规则：长辈只能确认自己的报告；家属必须具有有效绑定和 `REPORT_CONFIRM` 授权。`ACCEPTED` 将报告置为 `CONFIRMED`、订单置为 `COMPLETED`；`REJECTED` 将报告置为 `REJECTED`、订单退回 `WAIT_REPORT`。归档建议决策更新已有 `health_info_review_task`，不创建数据库定义之外的数据。

## 测试

`UserSidePhaseApiTest` 覆盖阶段 6/7/9/18 核心链路，`Phase16ReportAckApiTest` 覆盖阶段 16：

- 家属发起绑定、长辈确认、家属调整授权、家属撤销。
- 家属读取和修改已授权长辈档案。
- 家属维护服务地址和默认地址。
- 护士访问家属绑定接口被拒绝。
- 家属不能通过长辈确认接口代替长辈确认绑定。
- 健康检查返回阶段 1 契约字段。
- 管理员读取演示数据状态，护士被拒绝。
- 长辈接受报告后报告与订单状态同步。
- 家属驳回报告后订单退回待生成报告。
- 家属选择归档建议后审核任务分别进入通过或驳回状态。
- 无 `REPORT_CONFIRM` 授权的家属返回 403。
